data communications

it depends on four characteristics:

1. Delivery : 정확한 목적지에 전달, 원하는 장치나 사용자에게 전달
2. Accuracy : 정확하게 전달
3. Timeliness : 적정시간 내에 전달
4. Jitter : 음성이나 동영상과 같은 실시간 통신에서 패킷 도착 시간의 불일정에 따른 
품질저하 요인

순서대로 룰이 실행되어야함
메세지만 보내는 행위 뿐만 아닌 부수적인 절차가 존재해야함
이런것들을 프로토콜이라고 부름
 
프로토콜 - 통신을 잘하기 위한 규칙들이 순서대로 적혀있는 하나의 집합,
               절차를 포함한 통신 규칙
	      데이터 통신을 통제하는 규칙들의 집합(상호 합의)
프로토콜 자체는 일상용어임

★rule과 procedures가 중요 용어임
규칙과 절차가 가장 중요(프로토콜이 무엇인지 정의하라고 시험에 냄)
(규칙과 절차라는 단어가 중요함)

데이터가 흘러가는 관점에서 세가지 방법
simplex
half-duplex : 반이중, ex)무전기(동시에 대화는 불가능, 듣거나 말하거나 한가지씩만 가능)
full-duplex : 전이중, ex)요즘 핸드폰(동시 대화 가능)


네트워크
device, node : 통신
: device를 묶은 것

직접연결될경우 전선이 많이 필요함
한 군데(ex, 공유기, hub)로 다 연결시켜서 연결할 경우 부수적인 정보가 필요함

통신에 필요한 모든 device는 네트워크라고 칭함

네트워크를 만드는 방법
어떤 네트워크가 좋다 나쁘다를 판단할수 있는 기준이 필요함

performance(성능) : 전송시간, 응답시간, 처리율, 지연 
reliability(신뢰도) : 손실이 발생할 경우 얼마나 발생하는가, 복구하는데 얼마나 걸리는가
얼마나 튼튼한가
실시간의 경우 약간의 오류는 무시하고 넘어감, (신뢰도<속도)
security(보안) : 허가받지 않는 자들이 접근하는 것에 대해 얼마나 잘 보호하는가

물리적 구조
a point to point : 일 대 일
채널의 전체 용량은 두 기기간 전송을 위해 사용

multipoint : 일 대 다수
3개 이상의 특정 기기가 하나의 링크를 공유하는 방식

Topology : 접속형태(시험에 많이 나옴), 네트워크에서 노드와 링크의 물리적인 연결형태


mesh(그물)형 topology
모든 노드를 다 직접연결
n*(n-1)/2
n-1개의 입출력(I/O) 포트

- 장점
원활한 자료 전송 보장
높은 안정성
비밀 유지와 보안
결함 식별과 분리가 비교적 용이
신뢰도가 높아짐

- 단점
케이블의 양과 요구되는 I/O 포트 수
설치와 재구성의 어려움


star(성)형 topology
hub에 device들을 연결
허브(hub)라는 중앙제어장치(central controller)와
전용 점대점 링크 구성
각 장치간 직접적인 통신 불가
모든 전송은 제어 장치를 통해 전송
1개의 채널, 1개의 I/O 포트가 요구됨

장점
그물형 접속형태보다 적은 비용
설치와 재구성이 용이함

단점
허브가 고장나면 전체 시스템 고장


bus(버스)형 topology
하나의 케이블에 연결된 모든 device에 데이터 전송
통신 오류가 생김
동시에 데이터를 보낼경우 충돌이 생겨 오류가 남
탭(tap)과 유도선(drop line)에 의해 버스에 연결

장점
설치가 쉽다.
가장 적은 양의 케이블 사용

단점
재구성이나 결함 분리가 어려움
중추 케이블의 결함시 다수의 장치에 영향을 줌

돈을 아낄 수 있는 버스나 스타형을 많이 씀


ring(링)형 topology
각 장치는 repeater(중계기)를 통해 재전송함
토큰을 가지고 있을 경우 데이터 전송이 가능

이런 방식으로 데이터를 전송함
받아서 검증후 다음 device로 전달함

장점
설치와 재구성이 쉽다
신호는 항상 순환
경보(일정 시간 내 신호가 수신되지 않을 시 위치를 경보) 사용

단점
단방향의 경우 링의 결함 시 전체 네트워크 마비


네트워크 타입
LAN(Local-area network), WAN(Wide-area network)
도시하나정도 크기 : MAN
크기로 영역구분 : 작음 <- BAN, PAN, LAN, MAN, WAN -> 큼

LAN(근거리 통신망)
개인소유 또는 단일 사무실, 건물 혹은 학교 캠퍼스 등에 있는 장치들을 서로 연결하여
자원 공유를 목적으로 설계
버스, 링, 스타 형 사용

WAN(광역통신망)
국가, 대륙 또는 전세계를 포괄하는 광대역 영역에 데이터, 음성, 영상 및 비디오
정보의 장거리 전송 제공
거리제한 X
통신회사가 임대를 목적으로 만들어 사용



회사A - 라우터 - 회사B
internetwork = internet
네트워크와 네트워크를 연결
라우터들의 네트워크 - WAN

라우터와 라우터 사이의 전선의 용량은 커야함 그래서 대부분 광케이블임

backbone

라우터는 소프트웨어가 데이터를 보고 분석하여 연결
스위치는 전기적으로 연결함, 속도는 더 빠름


인터넷(Internet)
백본, 제공자, 네트워크, 소비자 네트워크로 구성
백본은 Sprintm Verizon, AT&T, KT, SKT, NTT같은 통신사 소유
백본망은 대등점(peering point)이라는 교환 시스템에 의해 연결
제공자 네트워크는 요금을 지불하고 백본 이용
소비자 네트워크는 인터넷 말단으로 인터넷에서 제공되는 서비스 이용
백본과 제공자 네트워크는 인터넷 서비스 제공자(ISP)라고 함
제공자 네트워크는 국가 또는 지구 인터넷 서비스 제공자(ISP)라고 함

ISP(시험에 많이 냄)
